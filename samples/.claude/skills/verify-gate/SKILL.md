# verify-gate

作る者(Maker)の主張ではなく、コマンドの合否(exit code)で品質を担保するための
3点ゲート。マージ前に各 worktree で実行し、緑になったものだけを親へ統合する。

## 手順
1. 変更のあったファイルを特定する(例: `git diff --name-only`)。
2. lint を対象ファイルに実行する。全角記号など規約違反を含めて赤があれば停止。
3. 型チェックを対象ファイルに実行する。赤があれば停止。
4. 関連するテストのみを実行する(スイート全体ではなく変更に関わる範囲)。赤があれば停止。
5. 3点すべて緑なら合格。1つでも赤なら原因票を添えて Maker へ差し戻す。

## 並列・直列の注意
- lint / 型チェックは worktree 毎に並列安全。
- テストスイート全体や DB・UI 依存の検証は直列レーンで1本ずつ実行する。

## 出力
- 合否(pass/fail)と、赤の場合は失敗したゲート名・要点(全ログは貼らない)。
