# loop-status

会話文脈ではなくディスク上の状態(WBS・decision_log・reports)を単一の真実として読み、
現在地を短く要約する。長い引用やログは貼らず、俯瞰だけを返す。

## 手順
1. WBS(例: `.claude/loop/WBS.md`)を読み、サブタスクの状態(未着手/進行中/ゲート通過/ブロック)を集計する。
2. state/(decision_log.md 等)と reports/ から、直近で決めたこと・棄却理由を拾う。
3. ゲート結果(3点ゲートの合否)を確認し、緑のもの/赤のものを分ける。
4. ブロック要因(依存待ち・承認待ち・直列レーンの占有)を列挙する。

## 出力(短く)
- 稼働中のループ/サブタスク一覧と各状態
- WBS 進捗(完了 / 全体)
- ゲートを通ったもの
- ブロック中のものと理由
- 次の一手(1つ)
