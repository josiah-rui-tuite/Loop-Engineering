# 人間ゲート ― ダッシュボード受入チェックリスト

配置先: `tools/templates/ui_human_gate.md`

前段の機械ゲートが全項目 PASS になってから使う(ゲートは two-stage-check の手順3で自分の題材向けに作る)。
コントラスト比、合計と内訳の整合、単位と時刻の有無は機械が見た。ここには載せない。
ここに残すのは、機械が満点を出せないもの、つまり人しか判定できないものだけである。

回答は「はい」か「いいえ」で書く。「おおむね良い」「見やすいと思う」は回答にならない。
**いいえが 1 つでもあれば不合格。** その項目だけを Maker へ差し戻し、次の周回に回す。

判定者は作り手ではなく、その画面を毎日使う現場の代表とする。

| # | 問い | はい / いいえ | いいえの場合、どこが |
|---|------|--------------|--------------------|
| 1 | 異常が起きている状態の画面を開いたとき、最初に目が入ったのは異常のカードだったか。 | | |
| 2 | 画面を 3 秒見ただけで、ラインを止めるべきか続けるべきかを判断できたか。 | | |
| 3 | 実際に立つ位置(画面から約 2m)から、主要な数値を読めたか。 | | |
| 4 | 現場の実際の照明(日中の直射光と夜勤の暗さ)の両方で読めたか。 | | |
| 5 | 視線が重要な順に流れたか。読み終わってから上に戻って探し直した箇所はなかったか。 | | |
| 6 | 一番大きく表示されている数値は、現場が一番知りたい数値と同じだったか。 | | |
| 7 | 色の意味を、凡例を見ずに言い当てられたか。 | | |
| 8 | 今日見なくてよい情報が視界に入って、見たい数値の邪魔をしていなかったか。 | | |
| 9 | 数値の更新が止まったとき、止まっていることに気づけたか。 | | |
| 10 | 朝礼で実際に使ってみて、口頭の補足説明を足さずに済んだか。 | | |
